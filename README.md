# sout-app — Специальная оценка условий труда (СОУТ)

Веб-приложение для специальной оценки условий труда (СОУТ): ввод результатов
измерений по производственным факторам, расчёт класса (подкласса) условий труда
по **Приказу Минтруда России № 33н** (методика СОУТ) и итоговая оценка рабочего
места.

Все вычисления выполняются на бэкенде; фронтенд — серверный HTML (Jinja2).

## Быстрый старт

Требуется Docker. Из каталога проекта:

```bash
docker compose up -d --build
```

Откройте <http://localhost:5680>. База содержит только справочники; организации и
рабочие места добавляются вручную через интерфейс.

### Остановка

```bash
docker compose down        # остановить сервис
docker compose down -v      # + удалить данные (том)
```

### Доступ по IP

Сервисы слушают на `0.0.0.0`, поэтому доступны по IP машины
(`http://<IP>:5680`). На Windows может потребоваться правило брандмауэра:

```powershell
New-NetFirewallRule -DisplayName "SOUT 5680" -Direction Inbound -Protocol TCP -LocalPort 5680 -Action Allow
```

## Классы условий труда

Хранятся целыми кодами: `1`, `2`, `31` (3.1), `32` (3.2), `33` (3.3), `34` (3.4),
`4`.

> ⚠️ Код класса 4 (опасный) численно **меньше**, чем 31–34, поэтому штатный `max()`
> к кодам неприменим. Для выбора наиболее тяжёлого класса используется хелпер
> `kut_max()` (`app/calculations/utils.py`), ранжирующий по порядку
> `KUT_ORDER = [1, 2, 31, 32, 33, 34, 4]`.

Итоговый класс РМ (Приложение № 22): максимальный по тяжести среди факторов; при
3 и более факторах одного подкласса 3.x — повышение на ступень.

## Структура проекта

```
sout-app/
├── docker-compose.yml      # сервис sout (порт 5680)
├── Dockerfile
├── requirements.txt
└── app/
    ├── main.py             # точка входа, справочные API/страницы
    ├── database.py         # engine, init_db + миграция новых колонок (ALTER TABLE)
    ├── models.py           # ORM: справочники, таблицы измерений, Summary
    ├── seed.py             # наполнение справочников из фикстур
    ├── calculations/       # расчётные модули по каждому фактору + summary + utils
    ├── routers/            # orgs, workplaces, factors (CRUD + расчёт класса)
    ├── templates/          # Jinja2: база, организации, РМ, формы факторов, справочники
    └── fixtures/           # JSON-данные справочников
```
