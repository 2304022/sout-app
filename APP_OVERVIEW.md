# Аттестация 5.1 — Обзор приложения

## Что это

**Специализированная десктопная программа для проведения Специальной оценки условий труда (СОУТ)** в соответствии с российским законодательством (ФЗ-426 «О специальной оценке условий труда»).

- **Версия**: 5.1.1698
- **Платформа**: Windows 32-bit
- **Стек**: VB6 (.exe), MS Access Jet (.mdb), MS Word VBA (.dot), MS Excel (.xlt/.xltx), HTML/XSLT
- **Главный исполняемый файл**: `Аттестация-5.1/attstart51.exe`
- **Дополнительно**: `diag_att_ctls5.exe` (диагностика), `JETCOMP.exe` (сжатие БД), `reg_num.exe` (регистрация лицензии)

## Архитектура расчётов

```
Пользователь вводит данные измерений
        ↓
attstart51.exe (VB6)
        ↓
Читает DataBase.mdb + справочники (himia.mdb, vibr.mdb и др.)
        ↓
VBA-макросы в .dot-шаблонах применяют пороговые таблицы приказа №33н
        ↓
Вычисляет @fact (факт.), @norm (ПДК/ПДУ), @kut (класс условий труда)
        ↓
Генерирует XML → передаёт в XSL-шаблон → HTML/Word/Excel документ
```

Именование полей в XSL/XML:
- `@fact` — фактический уровень
- `@norm` — норматив (ПДК, ПДУ)
- `@kut` — класс (подкласс) условий труда
- `@U095` — расширенная неопределённость измерения
- `@time` — время воздействия, %
- `@bm` — тип расчётного показателя (например `ss_razd`, `kcss_sum`, `plvn_sum`, `doza`)

## Формулы классификации условий труда (Приказ №33н, Методика СОУТ)

### 1. Химический фактор (Приложение №1)

Кратность превышения = Сфакт / ПДК

| Группа веществ | Кл. 2 | 3.1 | 3.2 | 3.3 | 3.4 | 4 |
|---|---|---|---|---|---|---|
| 1–4 кл. опасности (общие) | ≤ ПДКмакс и ≤ ПДКсс | >1,0–3,0× | >3,0–10× | >10–15× | >15–20× | >20× |
| Остронаправленного действия | ≤ ПДКмакс | >1,0–2,0× | >2,0–4,0× | >4,0–6,0× | >6,0–10× | >10× |
| Раздражающие | ≤ ПДКмакс | >1,0–2,0× | >2,0–5,0× | >5,0–10× | >10–50× | >50× |
| Канцерогены, репротоксиканты | ≤ ПДКсс | >1,0–2,0× | >2,0–4,0× | >4,0–10× | >10× | — |
| Аллергены высокоопасные | ≤ ПДКмакс | — | >1,0–2,0× | >2,0–5,0× | >5,0–15× | >15–20× |
| Аллергены умеренноопасные | ≤ ПДКмакс | >1,0–3,0× | >3,0–15× | >15–20× | >20× | — |
| Противоопухолевые, гормоны, наркоанальгетики | авт. класс 4 без измерений |

**Эффект суммации** (смеси однонаправленных веществ, Приложение №8):

```
К₁/ПДК₁ + К₂/ПДК₂ + … + Кₙ/ПДКₙ ≤ 1  →  допустимые условия
```

При превышении суммы ≥ 1 класс определяется аналогично одиночному веществу.

### 2. АПФД (Приложение №10)

Дополнительно к ПДК рассчитывается:
- **ПН** (пылевая нагрузка) = Ссс × Т (мг × смен)
- **КПН** (контрольная ПН) = ПДК × Т_стаж (мг × смен за нормируемый стаж)

| Тип АПФД | Кл. 2 | 3.1 | 3.2 | 3.3 | 3.4 |
|---|---|---|---|---|---|
| Высоко/умеренно фиброгенные (ПДК ≤ 2 мг/м³) | ≤ ПДК, ≤ КПН | >1,0–2,0× | >2,0–4,0× | >4,0–10× | >10× |
| Слабофиброгенные (ПДК > 2 мг/м³) | ≤ ПДК, ≤ КПН | >1,0–3,0× | >3,0–6,0× | >6,0–10× | >10× |

### 9. Лазерное и УФ-излучение (Приложение №18)

**Лазер**:
```
≤ ПДУ₁ и ≤ ПДУ₂     → класс 2
>ПДУ₁, ≤ ПДУ₂       → класс 3.1 (хроническое воздействие)
>ПДУ₂, ≤ 10×ПДУ₂    → класс 3.2
>10×ПДУ₂, ≤ 100×ПДУ₂ → класс 3.3
>100×ПДУ₂, ≤ 1000×ПДУ₂ → класс 3.4
>1000×ПДУ₂           → класс 4
```

**УФ-излучение**:
```
≤ ДИИ  → класс 2
>ДИИ   → класс 3.1 (работа только в СИЗ)
```

# Реализация на Python (sout-app)

Веб-реализация расчётной части оригинала. Повторяет логику классификации
условий труда по Приказу №33н; десктоп-обвязка (генерация Word/Excel, СИЗ,
профриски, компенсации, медосмотры, экспорт в ФГИС, нормативная база) сознательно
не переносилась.

**Расположение**: `C:\repo\sout-app`

## Стек и развёртывание

| Компонент | Технология |
|---|---|
| Backend | FastAPI + SQLAlchemy 2.0 (ORM) |
| БД | SQLite (том Docker `sout_data`, файл `/data/sout.db`) |
| Шаблоны | Jinja2, server-rendered HTML (примитивный фронтенд) |
| Контейнеризация | Docker Compose, один сервис |
| Порт | **5680** (bind `0.0.0.0`, доступно по IP) |

Запуск: `docker compose up -d --build` из `C:\repo\sout-app`.
Все вычисления выполняются на бэкенде; справочники наполняются из JSON-фикстур при
первом старте (идемпотентно). Схема БД создаётся через `create_all`, новые колонки
доустанавливаются лёгкой миграцией `_migrate_add_columns` (ALTER TABLE, без потери
данных).

## Кодирование классов и итоговая свёртка

Классы условий труда хранятся целыми кодами:
`1, 2, 31 (3.1), 32 (3.2), 33 (3.3), 34 (3.4), 4`.

⚠️ **Важно**: код класса 4 (опасный) численно *меньше*, чем 31–34, поэтому
встроенный `max()` нельзя применять к кодам напрямую. Тяжесть класса задаётся
порядком в `KUT_ORDER = [1, 2, 31, 32, 33, 34, 4]`; для выбора наиболее тяжёлого
класса используется хелпер `kut_max()` (`calculations/utils.py`), ранжирующий по
этому порядку. Применяется везде, где сворачиваются несколько классов: агрегация
измерений по фактору, объединение показателей внутри фактора и итоговый класс РМ.

Итоговый класс (`summary.py`, Прил. №22): максимальный по тяжести среди факторов;
при 3 и более факторах одного подкласса 3.x — повышение на ступень.

- Исправлен системный дефект свёртки классов (numeric `max` → `kut_max`).
